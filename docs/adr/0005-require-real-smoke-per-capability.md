# Require focused tests and real smoke per capability

A Website Capability is verified only when its command, documentation, focused tests, Adapter Contract, and redacted real-browser Smoke Evidence all pass. Implementation proceeds in order: read-only reconnaissance, minimal Synthetic Fixture, failing focused test, minimum passing implementation, documentation and contract, real smoke, then independent verification. Focused tests never depend on network access, private response recordings, or full-page HAR data. Every canonical capability requires its own smoke run, while parameter combinations may be covered by documented equivalence classes; unavailable login, permissions, or risk-control leaves the capability blocked rather than allowing code-only completion. Read evidence is fresh for 90 days, write and high-risk UI evidence for 30 days, and observed site drift invalidates related evidence immediately; stale evidence preserves implementation history but cannot support a new release decision.
