# Deliver reads first and gate website writes

The capability program catalogs both reads and writes, but delivers Read Capabilities before Write Capabilities. Every Write Capability remains non-mutating by default and verifies resulting website state. Reversible writes require `--execute`; communication, publication, and destructive writes also require a Preview Hash bound to the reviewed inputs; batch writes declare a maximum action count and report each result. Each write declares an Idempotency Policy: uncertain non-idempotent sends stop for remote-state verification and user direction, while state-setting writes may retry only after reading current state; item checkpoints prevent successful batch entries from replaying. The Foundation Wave provides the shared safety contract, then existing ChatGPT `ask` and `batch-ask` become its first tracer. General Write Wave work begins only after all 14 current sites reach Site Read Completion and the tracer is verified; a future website must reach its own Site Read Completion before admitting writes. Capability Verifiers test dry-runs only, and any live write smoke requires separate user authorization.
