# Parallelize across sites, not within one site

Foundation and shared-runtime work begins only from a recorded clean Execution Baseline and runs sequentially. Site implementation may run in parallel only across different websites, with one Site Owner per website and explicit File Ownership that excludes shared files; an independent Capability Verifier performs read-only review and real smoke. Parallel owners do not commit. A single Catalog Integrator serially checks scope, updates the Catalog Source and views, runs cross-site regression, and creates atomic ticket- or site-scoped commits after each Delivery Batch. Regressions revert the whole affected commit and its catalog status rather than leaving partial migration fixes; the Foundation schema cutover is a separate release boundary with its own complete rollback. Unknown or overlapping uncommitted changes block affected tickets and are never reset or absorbed by an executing model.
